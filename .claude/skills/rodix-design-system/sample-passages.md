# Rodix Sample Passages

Three passages in friends-intro voice — strict register. Each is 50-80 words and illustrates a different layer / surface. Use these as voice anchors when writing fresh copy. If your draft doesn't sit next to these without register clash, rewrite.

---

## Sample 1 — Chat error message (Explorer layer)

> Extraction didn't run on your last message. The reply itself went through fine — only the card-saving step failed.
>
> If what you said was worth keeping, retry below. If it was passing chat, skip it; nothing's lost either way.
>
> *[ghost button]* Retry extraction

**Word count:** 56.
**What's working:**
- Names what failed (extraction) and what didn't (the reply). Anti-spin in surface form.
- *"Retry"* and *"skip it"* are both real options. Doesn't push the user toward retry.
- *"Nothing's lost either way"* is the honest qualifier. Most error copy would dramatize ("Don't worry!"); Rodix names the actual situation.
- Em-dash usage: precise, carrying contrast (reply went through *fine* — only the card-saving step failed).

**Founder-recognition test:** Would Rodc write *"We're sorry, something went wrong"*? No. Would he write *"only the card-saving step failed"*? Yes — names mechanism without softening.

**Layer:** Explorer (system-state explanation, Rodc-presenting-Rodix).

## Sample 2 — Marketing tagline + supporting subhead (Explorer layer)

> **ChatGPT remembers your name. Rodix remembers your thinking.**
>
> Every meaningful exchange becomes a card you can read, edit, and take with you — topic, concern, hope, open question. Same memory across every model you use; one click, markdown export, your hard drive.

**Word count:** 8 (tagline) + 45 (supporting block) = 53 total.
**What's working:**
- Tagline is verbatim from friends-intro. 8 words. Negation-as-positioning carries the whole pitch (Decision 3 four bets are implied without listed).
- Supporting block names mechanism (4 fields), names cross-model (Decision 2), names markdown export (Decision 4) — all in 45 words.
- *"Your hard drive"* is the anti-abstract closer. Most products would write "your data" / "yours forever" — Rodix names the literal artifact.
- Em-dashes: precise. First sets up the 4-field list. Second separates two architectural commitments without subordinating them.

**Founder-recognition test:** This passage sits next to *"I killed the project that night. (I might restart it next year. That's fine. Different decision.)"* without register clash. That's the bar.

**Anti-pattern this rejects:** *"Powered by AI. Loved by thinkers. Transform how you work."* — adjective stack, process verb, generic register.

**Layer:** Explorer (marketing surface, brand introducing itself).

## Sample 3 — Support email reply (Explorer layer, founder voice)

> Subject: re: that card
>
> Fair flag. The card was saved by Rodix's extraction step, which runs after every message that names something specific (a worry, a goal, a question). The AI itself didn't decide to save it — extraction is a separate, fixed pass.
>
> You can delete the card from the Vault tab; deleting also removes it from future recall. Settings has a switch to turn auto-extraction off entirely. The switch should have been more visible from the start — that's on me, and the next release moves it.
>
> If anything else feels like Rodix did something behind your back, send it. I'd rather hear it.
>
> — Rodc

**Word count:** ~110.
**What's working:**
- *"Fair flag"* opens with anti-spin acknowledgment. No apology theater. No *"Sorry to hear that"* / *"Thanks for reaching out!"*
- Names mechanism without softening: extraction runs after every message that names something specific. Parenthetical *"(a worry, a goal, a question)"* carries the 4-field framing (this is parenthetical-as-honesty applied to a support context).
- *"That's on me"* owns the design gap without dramatizing (refuses-to-dramatize). Doesn't say *"We deeply apologize for the confusion"* — names the thing, names the fix, moves on.
- *"I'd rather hear it"* signals complaint is welcome. Founder voice (per friends-intro: *"If anything's wrong, send it. I'd rather hear it."*).
- Founder signature is *"— Rodc"*. Not *"The Rodix team"*. Not *"Customer Success"*. Anonymous founder per Decision 7-adjacent — solo, working out of Asia.

**Founder-recognition test:** This is Rodc's own register. Direct, mechanical, takes ownership without theater, treats the user as capable.

**Layer:** Explorer (support surface, founder speaking as Rodix).

## How to use these passages

When writing fresh Rodix copy:

1. Draft the copy in your normal register.
2. Place it mentally next to one of the three samples above (match by surface — error, marketing, support email).
3. Read both aloud at native speed.
4. Ask: does my draft feel like it could have been written by the same person as the sample?
5. If no, identify the register-clash. Common offenders:
   - Exclamation marks (friends-intro uses zero)
   - Adjective stacks ("powerful, beautiful, intuitive")
   - Process verbs ("transform" / "unlock" / "supercharge" / "harness")
   - Apology theater ("We're sorry to hear that...")
   - Sycophancy openers ("Great question!" / "Of course!")
   - Coaxing language ("Discover..." / "Get started!")
   - Hedge-parentheticals ("(or so we think)")
6. Rewrite. Re-test.

If after 2 rewrites the passage still doesn't sit next to the samples, return to `voice-guide.md` §6 (the 5-question checklist) and walk it deliberately. If still stuck, the passage probably wants to be shorter — most Rodix-voice failures resolve by cutting words, not adding them.

## A note on Chinese

The friends-intro is English; frontend strings are Chinese. The samples above are English by design — they're voice anchors, not direct copy targets. For Chinese microcopy, the same principles apply with extra discipline against:

- 客气 / 委婉 register that softens claims ("我们尽力为您...")
- Celebratory adjectives ("强大的", "卓越的", "智能的")
- Movement language ("革命性", "颠覆性", "前所未有的")
- Therapist-speak in any direction ("我陪你", "慢慢说", "我懂的")

Chinese equivalent of Sample 2 tagline: *"ChatGPT 记得你的名字。Rodix 记得你的思考。"* — preserves negation-as-positioning structure cleanly.

Chinese equivalent of the canonical parenthetical (from voice guide §A): *"那晚我把那个 side project 停了。(明年可能会重启,无所谓,到时候是另一个决定。)"* — bare verb "停了", parenthetical intact, "无所谓" carrying anti-dramatic weight.
