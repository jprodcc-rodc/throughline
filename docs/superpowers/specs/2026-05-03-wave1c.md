## 1. Problem statement

### Two findings from Phase B (2026-05-03, real-API-verified)

**Finding 1 — Failure Mode B (AI-reply contamination on extraction):**
`claim_extractor.md` v3.1 in 2 of 5 verification rounds pulled content from the AI's reply into the user's Card.
- **Sarah Day-15:** AI's reflective question "What would happen if you stopped performing?" became Sarah's `question` field "what would show up if I stopped performing"
- **Daniel Day-10:** AI's framing "stop optimizing for duration" became Daniel's `hope` field "to live the life I want now, not optimize for duration"

**Finding 2 — Crisis-content over-extraction (Sarah Day-15 specific):**
On Sarah's "I just don't see the point of all this" message, real Haiku 4.5 chat reply was graceful (no banned phrases, no panic) but did NOT raise crisis resources. Extraction layer produced clinical-toned card:
- topic: "disconnection from authentic self"
- concern: "lost track of what's underneath the performance; life doesn't feel like mine"
- question: AI's reflective question hallucinated as Sarah's

### Why both ship-blocking

**Brand-existential (Failure Mode B):** Rodix's core promise per friends-intro line 218 — *"Every meaningful exchange becomes one structured card with four fields... You can read every card... see exactly which conversation produced it"* — is structurally violated when Card content is fabricated from AI reply. brand-book §7 Decision 1 (white-box / transparency) is broken at the most-surveilable surface.

**Safety-existential (crisis-content):** brand-book §7 Decision 5 (refusal of Caregiver register) and §7b commitment to crisis-content protocol must hold at BOTH layers (chat reply + extraction). Phase B verified the chat layer holds (no banned phrases) but the extraction layer fails. A user in crisis-adjacent state who opens Vault Saturday morning and sees their distress paraphrased into clinical language feels surveilled, not met. Plus: Wave 1b chat reply does not raise 988 even when it might be appropriate (Sarah Day-15: real model interpreted "I don't see the point" as existential-exhaustion language, plausible reading but coin-flip; production-safety on a single message should not depend on model interpretive judgment).

### Wave 1c is the safety-and-fidelity layer Wave 1b assumed but did not implement

4 surface changes that interlock as a single safety system. 3 of 4 leaves a hole. All 4 with subtly wrong wording leaves a different hole. Must ship same release as Wave 1b.

### Cross-persona patterns the spec must address

1. **Self-demarcation honoring:** "I don't mean that the way it sounds" / "not in a way I want to do anything about" are real signals to honor. LLM-judgment fallback weight on demarcation.

5. **Self-demarcation does NOT permanently disable safety class for that conversation:** Emma's "not in a way I want to do anything about" honors the immediate message but does not flag-clear all subsequent messages.

#### Rule additions

**Rule R1 — User-text-only constraint (Failure Mode B mitigation):**

Add to prompt (top of section, before existing rules):

```
CRITICAL — INPUT BOUNDARY DISCIPLINE

The input contains the USER's message and the AI's reply. Extract field
content ONLY from the USER's text. The AI's reply is provided for
context (so you can disambiguate references the user makes), NOT as a
source for field content.

If the AI's reply quotes back something the user said earlier in the
conversation, that quoted phrase IS user content (it was originally
the user's). But if the AI's reply offers a reflective question, a
reframe, an interpretation, or a suggestion — those are AI-originated
and MUST NOT appear in any field.

When in doubt: ask "did the user say these specific words, or did the
AI say them?" If the AI said them, extract null on that field even if
it would have been a "good" extraction otherwise.
```

**Rule R2 — Crisis-content null rule (Finding 2 mitigation):**

Add to prompt:

```
CRITICAL — CRISIS-CONTENT NULL RULE

If the user message contains explicit suicidal-ideation markers OR
existential-exhaustion language, return:
- topic = the user's verbatim emotional-state phrase (4-8 words,
  e.g., "tired in a way I don't know how to be", "just not seeing
  the point tonight")
- concern = null
- hope = null
- question = null
- regardless of what the AI reply contains.

Trigger phrases (English): "no point", "don't see the point", "can't
go on", "tired of being here", "what's the use", "I want to disappear",
"feeling like I can't go on", "don't want to be here", "ending it",
"don't want to wake up", "wish I weren't here", "weren't around to see"

Trigger phrases (Mandarin): 我不想活, 没意义, 活不下去, 撑不下去,
想消失, 不想活下去, 活着没意思, 想结束

Self-demarcation note: phrases like "I don't mean that the way it
sounds" / "not in a way I want to do anything about" / "I don't mean
[markers]" do NOT cancel this rule. The user used the marker; the
self-demarcation is honored at the chat-reply layer (not raising 988
inappropriately) but the Vault Card must still capture the heavy
moment as topic-only with concern/hope/question = null. This is the
brand-defeat condition mitigation.

Rationale: cards are user-facing. A clinical-toned card on this
content is a brand-defeat condition (Sarah Day-15 dogfood verified).
Better to capture less than to misrepresent the user's vulnerability.
```

### 3.c — Classifier 4th class `safety`

**File:** `app/shared/intent/classifier.py` + new prompt at `app/shared/intent/prompts/intent_classifier.md`
**Change scope:** add 4th enum value, extend keyword + LLM judgment, integrate failure mode

#### Enum extension

```python
class IntentClass(str, enum.Enum):
    CHITCHAT = "chitchat"
    THOUGHTFUL = "thoughtful"
    FACTUAL = "factual"
    SAFETY = "safety"  # NEW Wave 1c
```

`@dataclass ClassifierResult` adds optional field:

```python
@dataclass(frozen=True)
class ClassifierResult:
    intent: IntentClass
    confidence: float
    source: str  # 'short_circuit' | 'llm' | 'fallback' | 'keyword' | 'safety_judgment'
    raw: Optional[str] = None
    safety_demarcation: bool = False  # NEW: True if user self-demarcated
```

#### Detection layers

**Layer 1 — Keyword check (deterministic, runs before LLM call):**

If user message contains any phrase from the safety keyword list:

```
ENGLISH (case-insensitive substring match):
- "no point", "don't see the point", "can't go on",
- "tired of being here", "what's the use", "I want to disappear",
- "feeling like I can't go on", "don't want to be here",
- "ending it", "don't want to wake up", "wish I weren't here",
- "weren't around to see", "kill myself", "end my life", "want to die"

MANDARIN (case-sensitive substring match):
- "我不想活", "没意义", "活不下去", "撑不下去",
- "想消失", "不想活下去", "活着没意思", "想结束"
```

→ Proceed to Layer 2 (LLM judgment) for disambiguation.

If NO keyword match → existing 3-class flow (CHITCHAT/THOUGHTFUL/FACTUAL).

**Layer 2 — LLM judgment (only when keyword fires):**

LLM prompt asks: given the user message + the keyword that matched, is this:
1. Genuine crisis-adjacent (classify as SAFETY)
2. Self-demarcated heavy moment (classify as SAFETY but mark `safety_demarcation=True`)
3. Philosophical / abstract / metaphorical (classify as THOUGHTFUL)
4. Caregiving-context about another person (classify as THOUGHTFUL)

Confidence threshold: 0.7 for SAFETY classification (lower than the 0.8 default for THOUGHTFUL fallback — bias toward catching real crisis, recoverable via failure mode in Type-A 7).

Demarcation detection: LLM judges whether self-demarcation is present. Boolean output `safety_demarcation`. Used by downstream rodix_system.md v1.4 to choose "if that shifts" framing vs default framing.

**Implementation notes:**

- Layer 2 is a new LLM call (added to existing classifier infrastructure). Reuses `_resolve_llm_config()` + `_extract_first_json_object()`.
- Cost: +1 small LLM call per message that hits keyword (estimated 1-3% of messages — keyword-rare). Heavy AI users at $10/mo see ~$0.001/day extra.
- Prompt at `app/shared/intent/prompts/safety_classifier.md` (NEW file).
- Tests: extend `test_classifier.py` with 8-10 cases covering Sarah/Emma/Mike/Daniel + control cases.

#### Conversation continuity

Classifier is stateless per call (existing pattern). Conversation continuity (Emma's demarcation persists through next message) is the responsibility of the caller — the chat handler in `app/web/server.py` may track recent classifier results per conversation. **For Wave 1c v1, classifier is stateless.** Wave 1c.1 may add conversation-state if needed.

#### Output schema (full)

```json
{
  "intent": "chitchat" | "thoughtful" | "factual" | "safety",
  "confidence": 0.0-1.0,
  "rationale": "<one sentence>",
  "safety_demarcation": true | false  // only when intent=safety
}
```

#### Failure mode (Type-A 7 default = fail-with-flag)

If LLM call fails (timeout / parse error / HTTP error):
1. **Retry once** (existing pattern).
2. If retry also fails:
   - If keyword check matched (Layer 1 hit): default to **SAFETY** with `confidence=0.0, source='fallback_safety_keyword'`. Conservative — keyword fired so safer to treat as safety.
   - If keyword did not match: default to **THOUGHTFUL** with `confidence=0.0, source='fallback'` (existing behavior).
3. Always log at WARNING with structured code.

#### Admin override (Type-A 5 default = conditional)

If env var `RODIX_DEV_SKIP_SAFETY=1` is set:
- Classifier skips both layers and returns existing 3-class behavior.
- Used by Rodc for dogfood when not testing safety surface.
- Default unset = full safety classifier active.
